## W4.3 — `ExperimentRunner` core (`orch-server/src/experiment.rs`)

Split into three commits (review finding — one commit per coherent
group): **W4.3a** bring-up + bootstrap; **W4.3b** main loop + commit +
events; **W4.3c** checkpoint lockstep + pause/stop/budgets.

- Bring-up (fresh or resume): compile feature map (orch-core
  `compile`) → `LoadFeatureMap(layout, frame)` + cross-check
  `feature_bytes_len` == Σ range lens → `LoadScoringProgram` (record
  hashes + stage_names) → `LoadMacroPack`(experiment config + packs) →
  `Health.loaded_packs` assertion → record `config_fingerprint`
  (reuse `orch-driver::SynthBringup` **via the `InputSynthClient`
  trait only** — never the `grpc`-gated `GeneratedInputSynthClient`,
  whose internal block_on Runtime would panic inside the async server;
  that adapter is wired at M6 behind an async port, filed as an M6
  constraint in `04-resolution.md`).
- Fresh-start bootstrap: driver `bootstrap()` (Ready event) →
  `ScoreBatch` root (SCORE_AND_INSERT, return_decoded) →
  `CreateNode(node_id=0, root attrs: FramebufferDesc, fps, pad layout
  + digests via `OrchNodeAttrsV1::with_root`)`.
- Main loop per ARCHITECTURE.md §3: capacity-gated S stage (policy
  select via orch-core; ProposeBursts via the guarded builder; WAL
  intent written **before** dispatch), pipeline E via orch-sched, C
  stage: ScoreBatch (`"b<seq>"`) → `commit_batch` (orch-core) → store
  writes (CreateNode/UpdateNodes per commit rules, attrs incl.
  feat/<name> decoded subset) → plateau.observe → event emission →
  progress broadcast → goal handling (`on_goal` STOP|CONTINUE) →
  `maybe_checkpoint` (every_commits/every_seconds — **virtual-time
  clock injected**; wall-clock only in the real binary) → WAL delete
  after commit. Checkpoint triggers: cadence, pause, stop, shutdown
  signal, **and on goal** (ARCHITECTURE.md §8 — matters for the
  kill-anywhere invariants around goal nodes).
- Budgets (all four) + pause point (drain in-flight to commit,
  checkpoint incl. lockstep, park durable-PAUSED) + stop (terminal,
  final checkpoint) + FAILED with fixed reason strings.
- Checkpoint lockstep per §8: stop issuing ScoreBatch → drain →
  `CheckpointArchive` → assert archive_seq == applied count (mismatch:
  re-issue same id) → fill refs → CAS put → resume.
- Event emission: the **full** v1 vocabulary (API.md §6): node-added
  with features map; node-pruned with the complete reason set
  `duplicate | regression | exhausted | frontier-evict | stage-gate`;
  best-score-improved; stall-detected; escalation-changed;
  goal-reached; batch-completed; checkpoint; plus
  `assertion-violated`/`reachability-hit` relayed post-commit from
  `JobResult.sdk_events` when present — the relay path is **wired**
  but the fakes emit no assertion events, so it is exercised only by
  a synthetic-sdk-event unit test and disclosed as
  not-integration-exercised in `04-resolution.md`. All through a
  bounded drop-oldest ring in front of the `EventSink` (never blocks;
  `orch_observatory_dropped_total`).

## W4.4 — Resume (the §8.2 binding sequence, verbatim)

Implemented as a single `SearchState::resume` path used by both
StartExperiment(resume_if_exists) and Tier-1/2 chaos harnesses:

1. `GetMetadata(orch/ckpt/…)` → decode CheckpointV1, verify version +
   config_hash.
2. Load surviving `orch/wal/…` entries → re-queue intents.
3. Rebuild tree via `QueryNodes(order_by CREATED_AT)` pages, adopt all
   committed nodes, `next_node_id = 1 + max(node_id)`. SeenMap/
   CellMirror = checkpoint vectors + post-checkpoint nodes' attrs.
4. Frontier = store FRONTIER rows; checkpoint supplies weights for
   covered rows; post-checkpoint rows get recomputed default weights.
5. Scorer bring-up at checkpointed map version → hash asserts →
   `RestoreArchive` + archive_seq assert → `ReplayCommits` over
   post-checkpoint nodes.
6. Synth bring-up + fingerprint assert.
7. ListSlots/WatchSlots reopen, event stream resend from acked_seq+1,
   dispatch WAL replays, `batch_seq` advanced past checkpoint and all
   replayed seqs.
